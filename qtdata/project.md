# 量潮数据项目管理

## 项目风险

如果实际难度和预估不一致，要怎么处理？
外部：这个责任归属我们，所以我们得得客户个交代。但是这类任务不确定本来也就大，所以有情况变化客户是有理解的基础是。但是我们得交代清楚为什么，然后和客户协商交付排期。一般是不改价格、我们自己承担的，所以前面报价通常会有这部分预留。
内部：一个技术问题出现的时候，整个团队没办法形成准确的判断，这首先是一个什么问题，到底是没有遵循现有流程做好，还是超出了我们的现有流程。如果超出了现有经验和流程，这个问题到底在不在技术委的能力范围内，如果真的需要求助，我们到底是应该求助一个咨询还是一个外包，要提出一个怎么样的具体问题。

当项目实施团队的力量不足以解决问题时，如何求助外部资源？
- 爬虫类：一般是需要供应商。爬虫是一个萝卜一个坑，而且很多网站很难爬，要咨询爬过的人或者请高手修复。比如上次的 Google专利。
- 清洗类：一般是需要和客户深度沟通和协作的。这种数据都是客户弄来的，只有客户自己比较清楚数据的情况，我们要做就必须得从头了解起。所以建立内部沟通的习惯，对于高效地和客户沟通本身就是一种训练和准备。
- 精炼类：一般是需要团队讨论的。这种问题大部分都是大模型，可能需要我带团队一起商量算法怎么设计，我提出一些方向以后技术委和项目组一起确定方案并且反复迭代。
