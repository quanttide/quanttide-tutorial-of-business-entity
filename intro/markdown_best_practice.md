# Markdown 最佳实践

## 概述

本文档基于 [Google 文档风格指南](https://google.github.io/styleguide/docguide/style.html)，提供 Markdown 写作规范。

## 核心原则

| 原则 | 说明 |
|------|------|
| 简洁 | 删除冗余词汇，用词简洁 |
| 一致 | 全文术语统一，格式一致 |
| 主动 | 使用主动语态，直接表达 |
| 具体 | 提供具体示例，避免模糊表述 |

## 标题规范

### 层级结构

- 最多使用三级标题（`#` / `##` / `###`）
- 避免在标题中使用标点符号
- 标题应简洁，明确概括内容

### 大小写规范

使用句子大小写（Sentence case）：

| 类型 | 示例 |
|------|------|
| ✅ 推荐 | `## 如何创建项目` |
| ❌ 避免 | `## 如何创建项目` |

## 列表规范

### 无序列表

- 使用 `-` 作为列表标记
- 列表项以分号结尾或使用完整句
- 保持列表项风格一致

### 有序列表

1. 步骤类内容使用有序列表
2. 确保编号连续
3. 每个步骤完整表达

### 嵌套列表

- 缩进使用 2 个空格
- 嵌套层级不超过 2 级

## 代码块

### 语言标注

必须标注代码语言类型：

\`\`\`bash
git status
\`\`\`

\`\`\`python
def hello():
    print("Hello")
\`\`\`

### 行内代码

使用反引号包裹：`variable`、`function()`、`file.md`

## 表格规范

### 基本格式

| 列1 | 列2 | 列3 |
|------|------|------|
| 内容 | 内容 | 内容 |

### 规范

- 表头加粗
- 列对齐使用冒号（`:--`、`:--:`、`--:`）
- 内容简洁，避免单元格过长

## 链接规范

### 外部链接

[链接文本](https://example.com)

### 内部链接

使用相对路径：`[文档](./docs/guide.md)`

## 引用规范

### 块引用

> 使用块引用标注重要提示、警告或引用内容

### 内联引用

使用**粗体**强调关键术语。

## 常见错误

| 错误 | 正确 |
|------|------|
| 非常简单的 | 简单的 |
| 也就是 | 直接说明 |
| 这个那个 | 具体指代 |
| 使用xxx来进行 | 使用xxx |
