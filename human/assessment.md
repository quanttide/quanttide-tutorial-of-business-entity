# 考核方法

量潮考核分**市场序列**与**技术序列**，不考知识技巧，考判断力与思维结构。

## 市场序列——认知翻译能力

| 题目 | 考察点 |
|------|--------|
| 拆解高转化内容 | 分析能力 |
| 改写内部经验文章 | 翻译能力 |
| 规划系列内容 | 系统思维 |

| 维度 | 合格 ✓ | 不合格 ✗ |
|------|--------|---------|
| 改写能力 | 保留核心素材，自然植入转化 | 纯广告或几乎没改 |
| 叙事意识 | 有真实冲突和挣扎过程 | 成功学叙事，像营销 |
| 转化设计 | 信息出现水到渠成 | 硬推销或重复出现 |
| 用户视角 | 开头有代入感，用语通俗 | 用内部术语自说自话 |

**定级：** S=可直接发布 → A=质量好有决策逻辑 → B=方向对但落地偏差 → C=纯广告或几乎没改

## 技术序列——宪法/判例思维

| 题目 | 考察点 |
|------|--------|
| 识别硬编码假设 | 解耦直觉 |
| 设计抽象模型 | 宪法思维 |
| 设计自举机制 | 工程哲学 |

| 维度 | 合格 ✓ | 不合格 ✗ |
|------|--------|---------|
| 概念理解 | 区分共性与特性，视现有实现为判例 | 认为抽象是过度设计 |
| 抽象建模 | 属性通用有舍弃，有扩展点 | 模型过于具体或空泛 |
| 工程判断 | 每步有解释，主动声明假设 | 无解释，不考虑失败场景 |
| 沟通质量 | 结构清晰，假设显式 | 无重点，图表缺失 |

**定级：** S=三题出色模型干净 → A=前两题出色有取舍 → B=方向对但模型粗糙 → C=不理解为何要抽象

## 操作规范

| 阶段 | 要点 |
|------|------|
| 发题前 | 面试官先熟悉材料，确保候选人可获取所需信息 |
| 考核期间 | 2-3 天异步，只答事实性问题，观察提问质量 |
| 评估后 | 简短反馈，通过者产出直接作为入职起点 |
