# Vibe Writing：多模型协作写作工作流

## 核心理念

不同大语言模型有各自的性格特长。与其用一个模型包揽所有工作，不如让模型各司其职，发挥各自优势，形成批改与表达的循环。

### 模型分工

| 模型 | 角色 | 擅长 | 不擅长 |
|------|------|------|--------|
| **GLM（智谱）** | 评审专家 | 批判性思维、结构分析、逻辑审查 | 创意表达、文采修饰 |
| **DeepSeek** | 写作者 | 文字表达、语言流畅、风格调整 | 自我审查、发现盲点 |

### 工作流

```
初稿 → GLM 评审 → DeepSeek 重写 → GLM 再评审 → DeepSeek 再优化 → 定稿
```

每一次循环，文档质量都会上一个台阶。

## 实践案例

以《LLM 命令行工具使用指南》的编写为例，完整演示一遍流程。

### 第一轮：初稿

作者根据实际操作经验，快速写出一篇初稿，包含安装、配置、使用等内容。初稿的特点是快和真实，但结构和表达往往粗糙。

### 第二轮：GLM 评审

将初稿交给 GLM，以评审专家的角色给出结构化意见。

GLM 的评审意见通常涵盖三个维度：

- **结构**：章节顺序是否合理、层级是否清晰、内容是否有重复或遗漏
- **内容**：信息是否准确完整、示例是否有风险、时效性如何
- **表达**：标题是否精确、用词是否专业、排版是否易读

### 第三轮：DeepSeek 重写

将 GLM 的评审意见和原文一起交给 DeepSeek，让其按意见重构。

DeepSeek 擅长将结构化的评审意见转化为流畅的文档，同时保留原文的实操价值。

### 第四轮：循环迭代

如果 GLM 仍有改进建议，可以继续下一轮循环。通常 2-3 轮即可达到较高水准。

## 为什么这样分工有效

### GLM 的批判性思维

GLM 系列模型在逻辑推理和结构化分析方面表现突出。它善于：

- 发现文档中的逻辑漏洞和前后矛盾
- 识别章节之间的层级关系是否合理
- 指出示例代码中的潜在风险
- 给出条理清晰的改进建议

就像一个严格的审稿人，不放过任何一个问题。

### DeepSeek 的表达能力

DeepSeek 系列模型在文字表达和内容重构方面表现突出。它善于：

- 将评审意见转化为具体的文档修改
- 保持原文核心信息的同时优化结构
- 使用更专业、流畅的语言重新表述
- 补充合理的过渡和衔接内容

就像一个经验丰富的编辑，能把粗糙的初稿打磨成成品。

### 互补效应

单一模型自审自改时，容易陷入思维盲区——它写的内容它自己觉得没问题。而引入另一个模型进行外部评审，相当于引入了外部视角，打破了这种盲区。

这正是 Vibe Writing 的核心：让不同的 vibe 相互碰撞，产生比单一模型更好的结果。

## 适用场景

| 场景 | 评审模型 | 写作模型 | 说明 |
|------|----------|----------|------|
| 技术文档 | GLM | DeepSeek | 结构严谨、表达清晰 |
| 产品文案 | GLM | DeepSeek | 逻辑清晰、语言流畅 |
| 学术论文 | GLM | DeepSeek | 论证严密、表达规范 |
| 创意写作 | DeepSeek | GLM | 角色可以互换 |

## 自动化脚本

可以将整个流程编写为脚本，一键执行。

```bash
#!/bin/bash
# vibe_write.sh — 多模型协作写作

INPUT="$1"
REVIEWER="glm-5"
WRITER="deepseek-v3.2"
ROUNDS="${2:-2}"

echo "=== Vibe Writing 开始 ==="
echo "评审模型: $REVIEWER"
echo "写作模型: $WRITER"
echo "迭代轮数: $ROUNDS"
echo ""

current="$INPUT"

for i in $(seq 1 $ROUNDS); do
    echo "--- 第 $i 轮 ---"

    # GLM 评审
    echo "评审中..."
    review=$(llm -m "$REVIEWER" "请作为技术文档评审专家，对以下文档进行评审，指出结构、内容、表达上的不足，并给出改进建议。请条理清晰地列出评审意见。

---

$(cat "$current")")
    echo "$review" > "review_$i.md"
    echo "评审意见已保存: review_$i.md"

    # DeepSeek 重写
    echo "重写中..."
    llm -m "$WRITER" "请根据以下评审意见重构文档。要求结构清晰、内容准确、语言专业，直接输出完整的 Markdown 文档内容，不要包含多余的解释。

【评审意见】
$review

【原文档】
$(cat "$current")" > "draft_$i.md"
    echo "重写稿已保存: draft_$i.md"

    current="draft_$i.md"
    echo ""
done

echo "=== Vibe Writing 完成 ==="
echo "最终稿: $current"
```

使用方法：

```bash
chmod +x vibe_write.sh
./vibe_write.sh 初稿.md 3
```

## 注意事项

1. **评审意见要具体**：给 GLM 的 prompt 应明确要求从结构、内容、表达三个维度评审，避免泛泛而谈
2. **重写指令要清晰**：告诉 DeepSeek 直接输出完整文档，不要只给修改建议
3. **保留中间产物**：每一轮的评审意见和重写稿都值得保留，方便回溯和对比
4. **适时停止**：通常 2-3 轮后边际收益递减，不必过度迭代
5. **人工把关**：最终稿仍需人工审核，确保技术细节的准确性
