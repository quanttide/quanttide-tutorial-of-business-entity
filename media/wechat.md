# 如何运营量潮科技公众号？

这是一篇写给公司在职及候选新媒体运营的工作教程。

## 工具

我们目前的内容生产流程非常简单。只需要一个 Markdown 转公众号的网站即可。

内部资料大多要求以 Markdown 格式或者可以转成 Markdown 格式的飞书文档等形式。所以只需要把内容复制粘贴到网站输出，再到公众号后台黏贴上去就可以。

## 素材

目前公司可以公开的内容正在大规模迁移到 GitHub 组织 quanttide。如果没有需要直接发布的业务最新动态，那么大部分内容都可以直接从“quanttide-tech”及其关联的子仓库获取。

也有部分情况是随着业务动态，发布以后整理到 GitHub 对应仓库。比如本文来源于内部培训的信息总结。还有一些随着客户和业务活动遇到的问题，或者内部活动产生的进展和经验。

## 分组

目前的合集分为业务和资产两类。

业务类以品牌命名，通常为“量潮 xxx”。包括但是不限于“量潮数据”、“量潮课堂”、“量潮云”、“量潮咨询”、“量潮招聘”等。通常文章标题也会以“| 量潮 xx”结尾以显式分组。

资产类则以资产名称命名，通常为“工作 xxx”。包括但是不限于工作章程、工作手册、工作教程、工作历史、宣传册、工作报告、工作日志等。

## 选题

​市场比较喜欢看业务进展，通常数据好于其他内容，而且通常发了就会有转化。

在选题方面优先发布业务信息，刻意营销的反而容易限流，实打实地不经修饰地直接发布反而转化更好。

其次发布和业务信息间接相关的信息为业务宣传提供辅助佐证，包括但是不限于工作章程等。

创始人札记和社论等和业务关系较远的内容，比较适合在创始人的个人账号发布，数据通常会好于公司账户。

目前的工作重点是浙理校企合作的进展记录和案例介绍。业务归属量潮科技，资产归属分别为工作历史和宣传册。

从 “iGuo 说”的数据来看，有数篇文章阅读量在 500-1000，高于账号平均阅读量 50-100。我们推测市场很感兴趣我们的教育改革与校企合作。

从“量潮科技”的最新数据来看，转化率也不错。昨天我们发布了大数据实践课的任务书，没有发布定价信息，今天后台就已经收到了几个咨询。我们已经安排对接。

## 策略

公司账号最主要的指标是转化率。即类似于前文提到的后台表达意向，或者通过我们提供在新媒体上的渠道直接联系我们等。

围绕这个转化率，保证内容的时效性是关键，因为公司业务变化飞快，保证最新信息披露及时，就是保证获客与增长能力。

单篇文章内容在 500-1000 字读者友好，如果内容较多可以后台排队。也不需要过于纠结排版，只需要通过 Markdown 转微信公众号排版工具保持最基本的可读性即可。

内容的阅读量和转化率往往没有直接关系。比如，前文提到的任务书一天只有五十个阅读，但是已经有了几个自然转化，如果刻意设置转化可以增长两三倍以上。招聘吐槽贴虽然只有 100 个阅读，但是给招聘公告导流了几倍，招聘申请也因此翻了 5 倍。

历史数据是非常重要的参考，特别是 iGuo 说公众号的后台数据。有很多内容在创始人个人公众号上已经经过了验证可行，只需要根据最新情况加工总结即可。

## 角色

机构号运营的重点在于做好业务宣传的“发言人”和文化展示的“史官”两个角色。前者为第一优先级，后者为第二优先级。

作为发言人，及时披露公司最新动态，或者根据公司增长需求披露相关业务宣传资料。主要资产来源是公司宣传册及其等价物。

作为史官，在配合业务宣传的情况下披露业务发展历史、服务记录等工作历史，可以很好的展示我们的品牌形象。主要资产来源是工作历史及其等价物。
